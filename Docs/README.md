# Documentation

## Overview

This directory contains the technical documentation developed for the **ChairlessChair-IoT** project.

The documents included here describe the system architecture, engineering decisions, hardware design, firmware workflow, project cost analysis, and supporting diagrams used throughout the development of the wearable assistive system.

---

# Documentation Structure

| File                                                     | Description                                                                                                                             |
| -------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| **ProjectReport.pdf**                                    | Complete technical report covering project objectives, system design, implementation, testing, results, and future work.                |
| **BOM.md**                                               | Bill of Materials listing all hardware components, quantities, specifications, and estimated costs.                                     |
| **Engineering_Design_Decisions_&_Trade-off_Analysis.md** | Engineering decisions made throughout development, including component selection, design trade-offs, and system architecture rationale. |
| **System Architecture Diagram.png**                      | High-level architecture showing interactions between sensors, controller, motor drivers, actuators, power system, and IoT dashboard.    |
| **System Block Diagram.png**                             | Functional block diagram illustrating data flow between the major hardware and software modules.                                        |
| **Firmware Flowchart.png**                               | Flowchart describing firmware initialization, autonomous control, manual override, safety logic, and web server operation.              |
| **State Machine Diagram.png**                            | State transition diagram for actuator behavior, operating modes, and safety states.                                                     |
| **Power Architecture Diagram.png**                       | Electrical power distribution showing isolated power domains for the controller and actuator system.                                    |
| **Control Logic Diagram.png**                            | Control algorithm illustrating FSR signal processing, threshold-based intention detection, actuator control, and safety mechanisms.     |

# Purpose

These documents provide a comprehensive overview of the ChairlessChair-IoT project, including its mechanical, electrical, embedded, and IoT subsystems. They are intended to support:

* Technical documentation
* Engineering design reviews
* Academic project evaluation
* GitHub portfolio presentation
* Recruiter and employer review
* Master's and research applications
